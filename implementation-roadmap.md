## Стек

- Основная система: `TypeScript`
- Backend: `Node.js + TypeScript`
- Frontend: `TypeScript`
- База данных: `PostgreSQL`
- Очереди/кэш: `Redis`
- ORM: `Prisma`
- Контейнеризация: `Docker Compose`
- ML/R&D контур: `Python` как отдельный сервис

## Фаза 1. Foundation

Статус:

- [x] Завершено

Что сделать:

- собрать монорепо;
- поднять `api`, `web`, `worker`;
- подключить `PostgreSQL`, `Redis`, `Prisma`;
- сделать базовую схему БД;
- добавить миграции и seed;
- сделать базовый `auth`, `settings`, `audit logs`, `jobs`;
- собрать docker-запуск одной командой.

Как:

- `apps/api` как базовый HTTP API;
- `apps/web` как admin dashboard shell;
- `apps/worker` как runtime обработки background jobs;
- `packages/database` как слой доступа к БД;
- `Prisma` migration + seed;
- `docker compose up -d --build` для полного старта.

Результат:

- рабочий инженерный фундамент;
- API отвечает и работает с БД;
- worker обрабатывает jobs;
- dashboard поднимается;
- всё стартует через Docker.

Как протестить:

- `npm run typecheck`
- `npm run docker:up`
- открыть `http://localhost:4000/health`
- открыть `http://localhost:4000/platform/bootstrap`
- открыть `http://localhost:3000`
- открыть `http://localhost:4100`
- создать job через `POST /jobs` и проверить, что worker перевёл её в `completed`

## Фаза 2. Анализ индустрии и компетенций

Статус:

- [ ] Не начато

Что сделать:

- подключить первые источники вакансий;
- собрать ingestion вакансий;
- извлекать компетенции из описаний;
- хранить вакансии и связи vacancy-competency;
- построить матрицу соответствия программы и индустрии;
- добавить базовую визуализацию разрывов.

Как:

- начать с `HH` и одного дополнительного источника;
- ingestion запускать как background jobs;
- extraction сначала сделать rule-based + простые AI prompts;
- результат складывать в `vacancies`, `competencies`, `programCompetencies`;
- матрицу отдавать через API и показывать в dashboard.

Результат:

- в системе есть реальные вакансии;
- компетенции извлекаются автоматически;
- виден разрыв между программой и рынком;
- можно выбрать приоритетную индустрию.

Как протестить:

- запустить ingestion job;
- проверить, что вакансии появились в БД;
- проверить, что competency links созданы;
- открыть API матрицы соответствия;
- проверить, что dashboard показывает числа не из seed, а из ingestion.

Как протестить:

## Фаза 7. Память агента и адаптация

Статус:

- [ ] Не начато

Что сделать:

- хранить историю всех действий агента;
- анализировать успешные и неуспешные коммуникации;
- внедрить adaptive recommendations;
- подготовить отдельный ML-контур.

Как:

- сначала строить memory как event log + retrieval;
- затем добавить аналитику outcomes;
- потом вынести `Python` сервис для R&D;
- позже подключать `QLoRA` и более сложную адаптацию.

Результат:

- агент запоминает историю;
- рекомендации улучшаются на основе прошлых результатов;
- есть основа для self-improving behavior.

Как протестить:

- накопить события коммуникаций;
- проверить, что outcomes сохраняются;
- сравнить рекомендации до и после истории;
- проверить, что adaptive logic меняет follow-up или tone;
- отдельно прогнать тестовый ML pipeline.
